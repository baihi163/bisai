# 4 模型检验

本章依据后验检验输出（`model_validation_checks.csv`）与统一汇总表（`model_validation_summary.csv`），从求解有效性、结果合理性、敏感性与稳健性、以及与问题一协调优化模型及非协同基线模型的对比四个方面，对问题二生命周期协同调度模型及相关对照结果进行检验。

## 4.1 求解有效性检验

**检验目的** 确认数值求解或对账流程处于可接受最优（或自洽）状态，且目标函数重算值与求解器/报表口径一致，为后续物理解释提供前提。

**检验方法** 对问题二生命周期协同调度模型各单次运行，核对求解器状态与“最优”标签，并将重算目标与求解器目标之差的绝对值与容差（0.02 元）比较。问题一协调优化模型及非协同基线模型若无 MILP 状态文件，则将“求解器最优”类检验标为不适用；基线依据全周对账表进行目标行差分检验。

**结果分析** 问题二代表性运行（如 `final_smoke`、`model_check_p2`、`verify_base`、`verify_evhet` 等）均为最优，目标绝对差约 \(10^{-11}\) 量级，低于容差。问题一 `objective_abs_gap_small` 为通过。非协同基线模型为规则仿真，无 MILP 状态项，记不适用；对账差分为零。问题二权重扫描中，`scan_rows_objective_finite` 通过，未见目标非有限或错误状态计数异常。

**小结** 在现有导出条件下，协调类 MILP 求解与对账满足预设有效性；非协同基线模型以对账自洽为主，检验项与之区分，结论不误判。

## 4.2 结果合理性检验

**检验目的** 核查解是否满足非负性、互斥性及储能边界等工程约束，避免数值最优与物理可行脱节。

**检验方法** 由时序功率递推储能能量并与上下界比对；统计购、售电及储能充、放电是否于同一时段同时超过设定阈值；检验弃光、切负荷、建筑恢复等量的非负性。问题二中“车队总充与总放同时为正”的参考项因多车聚合，标为信息性不适用，不作为硬违反。

**结果分析** 问题二与问题一在抽检运行上，`ess_energy_within_bounds` 通过；购售互斥、储能充放互斥时段计数为零；弃光、切负荷及建筑恢复相关非负检验通过。非协同基线模型储能轨迹在参数界内，购售与储能互斥及弃光、未供电非负检验通过；无建筑恢复列，对应项不适用。

**小结** 在已实施检验范围内，协调模型与基线均未出现与物理解明显矛盾的系统性违规。

## 4.3 敏感性与稳健性检验

**检验目的** 考察权重等参数扰动下输出是否异常，以说明结论对关键假设的依赖程度。

**检验方法** 基于问题二权重扫描汇总（`p2_weight_scan` / `scan_verify`）：统计重算目标有限性与错误状态；对电动汽车退化成本随退化权重作 Spearman 秩相关检验（阈值如 \(\rho \geq -0.3\)），排除与直觉相悖的强负相关。

**结果分析** `scan_rows_objective_finite` 通过。`scan_trend_ev_deg_vs_weight` 通过（记录中 Spearman 相关系数为 1.0，满足阈值），权重通道行为可解释。

**小结** 当前扫描样本下，模型满足预设稳健性筛查，未见权重扰动引起的数值崩溃或反常单调。

## 4.4 与问题一协调优化模型及非协同基线模型的对比检验

**检验目的** 在统一字段框架下，从成本结构、能量收支及机制差异对照问题一协调优化模型、非协同基线模型与问题二生命周期协同调度模型，说明可衔接性与差异来源。

**检验方法** 读取 `model_validation_summary.csv` 各行，对比总目标、运行成本、碳成本、储能/电动汽车退化成本、购售电量及储能、车队充放电量等，并结合 `model_validation_checks.csv` 的约束满足情况解释。须说明：问题二行来自单次/子场景目录，与全周口径的问题一、非协同基线模型在时段与场景上未必一致，对比以结构与检验结论为主，绝对数值并列须注明口径。

**结果分析** 汇总表在各自 `run_tag` 下显示：非协同基线模型（`baseline_default`）总成本与购电量较高，车队无放电能量，且无与协同恢复惩罚同义的导出项，与“非协同、规则驱动”设定一致。问题一协调优化模型（`p1_ultimate_latest`）总成本低于基线（约 39467 元对约 46410 元），并出现显著的车队放电与储能吞吐，能量分配结构因协调优化而改变。问题二生命周期协同调度模型（如 `model_check_p2`）在给定单次场景下的目标与能量与全周算例不可直接比大小，但汇总字段完整，且检验中报告最优与求解时间，便于审计。

**关于非协同基线模型唯一 fail 的说明** 检验表中，基线仅在 `ev_departure_demand_met` 为 fail：`sessions_unmet = 2`，`n_sessions = 102`（约 1.96%）。该判据基于会话级满足标志，非舍入毛刺，反映规则型非协同基线模型在少数离站会话上未能达到与 MILP 硬约束等价的满足水平，属策略能力边界。论文中应如实披露：基线宜作参照，不宜无说明地等同于服务质量保证或物理可行上界；未满足占比小，不否定其作为整体对照的用途，但须点明上述结构性局限。

**小结** 问题一协调优化模型相对非协同基线模型呈现与假设一致的成本与结构差异；问题二生命周期协同调度模型在子场景口径下通过同一后验检验且输出完备；基线在离场需求上存在少量未达标会话，须在对比中单独说明其含义。

## 模型检验结论

综合两表：问题二生命周期协同调度模型在代表性运行上均为最优，目标重算与求解器一致，关键物理与运行约束检验通过；问题一协调优化模型在对账与功率互斥等检验上正常；权重扫描未见目标非有限或反常秩相关。与非协同基线模型相比，问题一在统一指标下呈现与“非协同—协调”假设一致的成本与能量结构差异；问题二因场景口径不同，不宜与全周基线简单比绝对值。非协同基线模型在离站需求满足上出现 2/102 会话未达标，属规则策略局限而非数值误差，文中应明确其适用范围。综上，在现有数据与检验设计下，模型输出满足预设求解有效性、结果合理性与稳健性要求，与问题一及非协同基线模型的对照在解释上自洽，可作为后续分析与扩展的基础。
