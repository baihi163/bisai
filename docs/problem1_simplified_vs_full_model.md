# 问题1：简化原型（`p_1_0.py`）与正式主模型对照说明

本文说明 `code/python/problem_1/p_1_0.py` 在问题1中的**定位**：它是 **协同调度的简化原型**，**不是**问题1的最终主模型。正式主模型以 **`src/problem1/coordinated_model.py`** 与 **`docs/problem1_coordinated_model.md`** 为准。

---

## 1. `p_1_0.py` 的建模假设（简化原型）

1. **单母线、有功平衡**：与正式模型一致，不计潮流与网损。
2. **确定性、多时段**：使用附件时序数据；步长 $\Delta t$ 与 `ess_params.json` 中一致（如 15 min）。
3. **电源与负荷**：
   - 光伏以**可用出力上界**减去**弃光**进入平衡；
   - 负荷为 **`total_native_load_kw`（或分项列求和）** 的标量序列，**不**拆成多栋建筑独立决策。
4. **固定储能**：充放电功率有上界；能量递推与 SOC 上下界来自 `ess_params.json`；可选用**单二进制 Big-M** 实现充放互斥（`--no-ess-mutex` 时退化为仅功率盒约束，可能同时充放，仅作近似）。
5. **电网**：购电、售电分别受**时变进口/出口上限**约束。
6. **柔性负荷（聚合）**：
   - 用单一 **`P_shift`** 表示相对基准的「移峰」量，**有界**（上界为各建筑 `max_shiftable_kw` 之和的聚合）；
   - **按日**施加 $\sum_{t\in\text{日}} P_{\text{shift},t}=0$（移峰填谷日内守恒）；
   - 用 **`P_shed`** 与惩罚系数表示可削减功率（与正式模型中「未满足负荷」或柔性形式**不同**）。
7. **目标函数**：购电成本 − 售电收益 + 弃光惩罚 + 削减惩罚；**不含** EV 退化、**不含**分项建筑柔性 $|ΔP|$ 的完整线性化形式（与主模型可逐项对齐时需改目标）。
8. **求解器**：PuLP + **CBC**（MILP 时可含储能二进制变量）。

---

## 2. 相比正式主模型省略了什么

| 维度 | 正式主模型（`coordinated_model.py` + 论文） | 简化原型 `p_1_0.py` |
|------|---------------------------------------------|----------------------|
| **EV** | 会话级到站/离站、$\chi_{v,t}$、充放电功率与能量递推、**V2B 资格**、离站最低电量 | **完全省略**（功率平衡中无 EV 项） |
| **建筑负荷** | 多栋 $P^{\mathrm{bld}}_{b,t}=P^{\mathrm{base}}_{b,t}+ΔP^{\mathrm{flex}}_{b,t}$，分项界与代价 | **总负荷 + 单一 `P_shift`**，或削减 `P_shed`；非分项 `ΔP_{b,t}` |
| **柔性代价** | $c^{\mathrm{flex}}_{b,t}\cdot\phi(|ΔP|)$ 或二次型；可选爬坡、能量中性 | 聚合移峰 + 日平衡；惩罚形式与主模型**不一致** |
| **缺电** | 可选 $P^{\mathrm{uns}}$ 与惩罚 | 用 **`P_shed`** 类比削减，**不等价**于论文中的 $P^{\mathrm{uns}}$ 定义 |
| **购售互斥** | 可选 $u^{\mathrm{grid}}_t$ | **未建** |
| **储能互斥** | 可选 $u^{\mathrm{ess}}_t$；实现细节可与 PuLP 版不同 | 单二进制 Big-M 或关闭互斥 |
| **求解器** | **Gurobi**（大规模 MILP/QP） | **CBC**（规模与性能上限不同） |
| **数据接口** | `data_loader.CoordinatedInputData`、EV 会话 CSV | 聚合读入，无会话表 |

## 3. 适合做什么 / 不适合做什么

### 适合

- **快速验证**：数据路径、时段数、功率平衡是否可行、CBC 能否在可接受时间内求解；
- **轻量实验**：无 Gurobi 许可证时的**趋势性**试算；
- **教学或草稿**：展示「购电−售电−储能−弃光−负荷调整」的联立结构；
- **与主模型并行**：作为「降维」对照，但需在文中**明确声明**非完整模型。

### 不适合

- 作为问题1 **唯一**的「协同调度优化模型」写入正文而不加说明；
- 替代正式主模型做 **EV、V2B、多建筑柔性** 的定量结论；
- 与 **非协同 baseline** 做严格公平对照（baseline 与主模型在 EV/规则上需与赛题一致，原型未覆盖 EV）；
- 要求与 **`problem1_coordinated_model.md` 符号一一对应**的审稿级复现。

---

## 4. 为什么不能直接替代问题1主模型

1. **赛题与论文叙事**：问题1强调 **微电网—电动车—建筑** 的**协同**；省略 EV 与分项建筑后，**「协同」内涵不完整**，无法支撑「车网互动 / 会话补能」类结论。  
2. **数学对象不一致**：正式模型中的决策向量包含 $P^{\mathrm{ev,ch}}_{v,t}$、$P^{\mathrm{ev,dis}}_{v,t}$、$E_{v,t}$、$ΔP^{\mathrm{flex}}_{b,t}$ 等；原型无法产生与之可比的最优解。  
3. **对照实验**：与 `src/problem1` baseline / 主模型对照时，若混用原型作为主结果，会造成**模型层级不一致**（前後假设不同）。  
4. **工具链**：主模型与数据预处理（`data_loader`）、结果导出（`result_exporter`）一致；原型输出字段与命名不同，**不可**直接当作主模型结果表使用。

## 5. 若要将 `p_1_0.py` 升级为正式主模型，最少还需补哪些模块

以下按「达到与当前 `coordinated_model.py` 同等建模内涵」的**最小增量**列出（不要求仍用 PuLP；主模型现用 Gurobi 更现实）。

1. **EV 会话数据与参数**  
   - 读入 `ev_sessions_model_ready.csv`（或等价表）：到离站 slot、到站能量、离站需求、充放电上限、**V2B 标志**。  
   - 构造 $\chi_{v,t}$（`build_ev_chi_matrix` 同类逻辑）。

2. **EV 决策变量与约束**  
   - $P^{\mathrm{ev,ch}}_{v,t}$、$P^{\mathrm{ev,dis}}_{v,t}$、$E_{v,t}$；停车时段功率界；非 V2B 强制 $P^{\mathrm{dis}}=0$；能量递推；到站/离站能量条件。

3. **功率平衡改写**  
   - 在单母线方程中加入 $\sum_v P^{\mathrm{ev,ch}}$、$\sum_v P^{\mathrm{ev,dis}}$（与论文 §7.6 一致）。

4. **建筑侧由聚合改为分项（若论文采用多栋）**  
   - $ΔP^{\mathrm{flex}}_{b,t}$ 每栋变量；盒约束；可选爬坡与能量中性；目标中柔性项与主模型一致。

5. **缺电与柔性语义统一**  
   - 明确 $P^{\mathrm{uns}}$ 或仅用柔性/削减之一，并与论文符号对齐；删除或改写与主模型冲突的 `P_shed` 语义。

6. **目标函数对齐**  
   - 购售、弃光、柔性、缺电惩罚与主模型一致；若问题2需要，再加退化成本项。

7. **求解器与规模**  
   - 全时段 × 多 EV × 二进制互斥时，**PuLP+CBC 往往不足**；需迁移至 **Gurobi**（或接受极长求解时间并验证可行性）。

8. **工程对齐（可选但推荐）**  
   - 复用或对接 `src/problem1/data_loader.py`、`result_exporter.py`，保证输入输出与主模型一致，便于写表与画图。

完成以上增量后，本质上即 **复现或合并入 `src/problem1/coordinated_model.py`**，而非在 `p_1_0.py` 上「小改」。实践中建议：**以主模型代码为唯一完整版，原型仅保留为轻量旁路**。
